# Security Policy

## Supported Versions

We actively maintain and provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 5.1.x   | ✅ Supported        |
| 5.0.x   | ❌ Not Supported    |
| 4.0.x   | ✅ Supported        |
| < 4.0   | ❌ Not Supported    |

We strongly recommend upgrading to a supported version to ensure ongoing security updates and protection.

## What to Include

Please provide as much detail as possible:

- Description of the vulnerability  
- Steps to reproduce  
- Affected systems or versions  
- Proof-of-concept (if available)  
- Potential impact  

---

## Response Timeline

We aim to maintain clear and timely communication:

- Acknowledgment: within 24–72 hours  
- Initial assessment: within 3–5 business days  
- Resolution: depends on severity and complexity  

---

## Responsible Disclosure Guidelines

We ask that you:

- Do not publicly disclose the issue until it is resolved  
- Do not exploit the vulnerability beyond proof-of-concept  
- Do not access, modify, or delete user data  
- Do not disrupt services  

---

## Safe Harbor

We support good-faith security research and will not pursue legal action against researchers who:

- Follow this policy  
- Act responsibly  
- Respect user privacy and system integrity  

---

## Bug Bounty (Optional)

We may offer rewards for valid, high-impact vulnerability reports.

Eligibility is based on:

- Severity of the issue  
- Quality of the report  
- Reproducibility  

Rewards are determined at our discretion.

---

## Out of Scope

The following are generally not considered valid:

- Issues in unsupported versions  
- Low-impact UI/UX bugs  
- Theoretical issues without proof  
- Automated scan results without validation  

---

## Recognition

We may acknowledge contributors in a public Security Hall of Fame (with permission).
